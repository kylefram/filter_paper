\chapter{Method}
The following section outlines the structure of the project, from theoretical formulation, to data acquisition, to lab testing, to deployment. 
The camera pipeline is first developed in a simulated environment, with spectral transmission functions acquired from producer specifications and camera sensitivity functions acquired in the same way. 
Thus, a full simulation of our camera can be deployed on multispectral image sets and tested in our filter selection harness.
Later, measurements of a subset of filters are taken in the lab, a measurement of camera spectral sensitivity function is also done in the same controlled lighting environment as the filter measurements, and captures are taken of a Preferred Memory Color Chart (PMCC) for quantitative analysis. 

\section{Mathematical Framework}
In our model, we have a single grey-scale camera (only a brightness channel). Thus, our estimated new camera sensitivities are a function of the intrinsic sensitivities of this single-channel camera multiplied by the filter transmittance.
Let $f_i(\lambda)$ respectively denote the $i$'th filter's spectral transmission function, and let $m_{c,i}$ denote the transformation weights for combining the $i$'th filter into color channel $c$ (e.g. to best approximate the XYZ color matching functions). 
Then the effective spectral camera sensitivity function $\theta_c$ can be written as:
\begin{equation}
\theta_{c}(\lambda)= \sum_{i \in I} m_{c,i}{f}_i(\lambda)\sigma(\lambda)
\end{equation}
where $I$ denotes the set of $k$ indices from our set of $N$ filters and $\sigma(\lambda)$ denotes the sensor's spectral sensitivity function.

In the optimizations that follow, we will solve for the R, G and B sensors that best approximate the actual X, Y and Z color matching function sensitivities, respectively referred to as $x_c \in \{X,Y,Z\}$.

\section{Problem Formulation}

For approximations of XYZ, we follow the work of Hardeberg \cite{Filterselection} in adopting MSE loss between target and simulation over the visible spectrum as the loss function and minimizing loss via search for an optimal $\boldsymbol{M}$, the $3 \times k$ matrix of $m_{c,i}$ transformation weights. For each color channel $c$ we have MSE loss $L_c$ as:

\begin{equation}
    L_c = \int_{\omega}\|{x_c}(\lambda)-\theta_c(\lambda)\|_2 d\lambda
    \label{eq:5}
\end{equation}

Given that we are limited by 1 nanometer steps in measurement, we discretize over $\omega$, taking $\omega_d = \{400,401...750\}$, and functions having arguments in brackets representing functions, for example $\theta_c[\lambda]$, evaluated over discrete domain. We take $MSE$ loss over $\omega_d$ to compute our total loss per color channel:

\begin{equation}
    L_c = \frac{1}{| \omega_d |}\sum_{\lambda \in \omega_d}\|{x_c}[\lambda]-\theta_{c}[\lambda]\|_2
    \label{eq:6}
\end{equation}

We consider loss from all color channels equally, and minimize $\sum_c L_c$ by choosing $\boldsymbol{M}$ for a set of $I$ `active' filters, each having transmission functions $f_i[\lambda]$ over $\omega_d$, thus comprising a $k$ length vector of functions.
Omitting the constant $\frac{1}{|\omega_d|}$, we have for each set of filters $I$:

\begin{equation}
    \argmin_{\mathbf{M}} \sum_c \sum_{\lambda \in \omega_d}\|{x_c}[\lambda]-\sum_{i \in I}m_{c,i}{f}_i[\lambda]\sigma[\lambda]\|_2
    \label{eq:7}
\end{equation}

We define $\boldsymbol{F}$ as the stacked and discretized $k\times 351$ matrix of $f_i[\lambda]\sigma[\lambda]$ measured at each $\lambda \in \omega_d$. 
And the discretized set of $3 \times 351$ XYZ functions as $\mathbf{X}$. 

When using a least-squares loss $L_d$, we can solve for $\boldsymbol{M}$, in closed form, using the Moore-Penrose pseudo-inverse \cite{greville1960some, BEUTLER1965451}, denoted $A^+$ for a matrix $A$. 
The unique $351 \times k$ Moore-Penrose pseudo-inverse solution $\boldsymbol{F}^+$ has the desirable property that while we may not exactly satisfy the Luther condition, it provides the closest possible approximation for a given set of filter indices $I$ \cite{macausland2014moore}:
\begin{equation}
    \boldsymbol{M} \approx \boldsymbol{X}\boldsymbol{F}^+
\end{equation} 

%Kyle: either expand the pseudo inverse or give a reference
Finally, as a point of detail we must alter this optimization if any of our filters have 0 transmission in a part of the assumed wavelength interval. 
The IR-cut filter only has nonzero transmittance in [400,650] nanometers. 
So, when this filter is used we can only optimize within this wavelength interval, and thus we redefine $\omega_d = \{400,401,...650\}$.%Note that for algorithmic stability of the pseudoinverse, we must truncate to $\lambda \in \{400,401...650\}$ when using the $NOIR$ filter. 

Further, we consider applications using loss functions $L$ and their respective $L_d$ which may not be the MSE loss. 
We consider SMAPE and log-loss functions amongst others, for which analytical solutions do not necessarily exist. 
For these, we use a gradient descent algorithm with initial guess informed by the Moore-Penrose pseudoinverse.

As an extension of the above, we model the noise of our camera using a modified MSE loss plus an error term. 
Following Hanji \cite{hanji2020noise}, we model camera photon registration as a poisson process. 
Borrowing the terminology, we take the radiance at each pixel $p$ as $\phi(p)$, having pixelwise radiance defined as $R(p)~Pois(\phi(p)t)$.
Then, with gain $g$ and exposure time $t$, he expected scene radiance and variance, or noise, are determined by
\begin{equation}
    E(R(p)) = \phi(p)tg 
\end{equation} 
\begin{equation}
    V(R(p)) = E(R(p))g 
\end{equation} 
Thus, our photonic camera noise, V(R(p)) is a linear function of exposure time. 
We provide a study in reconstruction quality as a function of 'shutter speed,' which refers to the duration of capture per filter. 
This is again implemented by defining $L = MSE + V(R(p))$, which can be achieved with a few steps of gradient descent from the MSE pseudoinverse solution.

\section{Filter Selection}

While the number of filters $N$ may be large, we ideally would posit a filter wheel with fewer filters.
Some works, such as Finlayson and Zhu \cite{finlayson2020designing} aim to construct optimized filters for camera and illumination spectifics. 
However, off-the-shelf gel filters provide a cost effective tool, if a good selection can be made. 
We take $k$, the number of filters chosen from the $N$ sized universe, as well as $L_d$, the discretized loss function as per the above.

The problem of filter selection and $\boldsymbol{M}$ computation are closely related. 
For non-brute force methods of filter selection, we effectively 'guess' the combination that might produce the best MSE after choosing $k$.
This lends itself to either exclusion or inclusion algorithms, the former we refer to as LASSO, the latter as PCA. 


\subsection{Brute Force Search}
In the naive solution, we find the optimal filter combination for every subset of $k$ filters (where $k$ is small) drawn from the $N$ filter set by selecting the combination with the lowest loss, considering all combinations of filters (there are nearly 10,000,000 of these for $k=5$ in the full Rosco Supergel set). 
In Figure 2 we plot the errors across all sets of 5 filters (multiplied by the camera sensitivity) from the worst to best performing quintuple. 
We compute the Moore-Penrose pseudo-inverse for each set of filters. 
Our implementation is parallelized over 8 NVIDIA L4 GPUs, with a combined pool of up to 192GB VRAM, enabling us to compute brute force searches of $k=5$ in approximately 7 minutes.

\subsection{PCA Analysis}

To find an approximate high-performing set of $k$ filters without brute force search, we use a greedy selection scheme closely related to Matching Pursuit \cite{mallat1993matching}. The method uses a low-dimensional subspace induced by the target functions and iteratively selects filters whose projections onto this subspace are large and as mutually ``orthogonal'' as possible.

Let $\mathbf{X}\in\mathbb{R}^{3\times d}$ denote the discretized target CMFs over $\omega_d$ (with $d=|\omega_d|$) and let $\mathbf{F}\in\mathbb{R}^{N\times d}$ denote the candidate filter-sensor rows, i.e. row $i$ is $f_i[\lambda]\,\sigma[\lambda]$. We form a truncated basis for the target subspace via an SVD of $\mathbf{X}^\top$ and then greedily select $k$ rows of $\mathbf{F}$ using orthogonalized residual norms.

\begingroup
\setlength{\textfloatsep}{6pt}
\setlength{\floatsep}{6pt}
\setlength{\intextsep}{6pt}
\begin{algorithm}[H]
\caption{PCA / Matching-Pursuit filter selection and weight estimation}
\label{alg:pca-selection}
\begin{algorithmic}[1]
\Require Target matrix $\mathbf{X}\in\mathbb{R}^{3\times d}$; candidate rows $\mathbf{F}\in\mathbb{R}^{N\times d}$; number of selected filters $k$.
\Ensure Selected indices $I\subset\{1,\dots,N\}$ with $|I|=k$ and weights $\boldsymbol{M}\in\mathbb{R}^{3\times k}$.
\State Compute thin SVD: $\mathbf{X}^\top = \mathbf{U}\boldsymbol{\Sigma}\mathbf{V}^\top$.
\State Set $r\leftarrow \min(3,k)$ and $\mathbf{U}_r \leftarrow \mathbf{U}(:,1\!:\!r)$.
\State Project candidates into target subspace: $\mathbf{G} \leftarrow \mathbf{F}\,\mathbf{U}_r$ \Comment{$\mathbf{G}\in\mathbb{R}^{N\times r}$}
\State Initialize $I\leftarrow \emptyset$ and orthonormal basis $\mathbf{Q}\leftarrow [\ ]$.
\For{$t=1$ to $k$}
    \If{$t=1$}
        \State $i^\star \leftarrow \arg\max_i \|\mathbf{G}_{i,:}\|_2$.
        \State $\mathbf{q} \leftarrow \mathbf{G}_{i^\star,:}/\|\mathbf{G}_{i^\star,:}\|_2$.
    \Else
        \State Compute projection onto current span: $\widehat{\mathbf{G}} \leftarrow (\mathbf{G}\mathbf{Q})\mathbf{Q}^\top$.
        \State Orthogonal component: $\mathbf{G}_\perp \leftarrow \mathbf{G} - \widehat{\mathbf{G}}$.
        \State Set scores $s_i\leftarrow \|\mathbf{G}_{\perp,i,:}\|_2$ and force $s_i=-\infty$ for $i\in I$.
        \State $i^\star \leftarrow \arg\max_i s_i$.
        \State $\mathbf{q} \leftarrow \mathbf{G}_{\perp,i^\star,:}/\|\mathbf{G}_{\perp,i^\star,:}\|_2$.
    \EndIf
    \State $I \leftarrow I\cup\{i^\star\}$; $\mathbf{Q} \leftarrow [\mathbf{Q}\ \mathbf{q}]$.
\EndFor
\State Form selected row-matrix $\mathbf{F}_I\in\mathbb{R}^{k\times d}$ by stacking rows $i\in I$.
\State Least-squares weights (MSE): $\boldsymbol{M} \leftarrow \mathbf{X}\,\mathbf{F}_I^{+}$.
\State \textbf{Optional:} if using a non-MSE loss $L_d$, refine $\boldsymbol{M}$ via gradient descent initialized at $\mathbf{X}\mathbf{F}_I^{+}$.
\end{algorithmic}
\end{algorithm}
\endgroup

In our implementation, the optional refinement step is applied when the loss is not purely MSE (e.g., SMAPE or noise-aware variants): we treat $\boldsymbol{M}$ as a learnable parameter, initialize it from the pseudo-inverse solution, and take a small number of Adam steps to reduce $L_d$.

To find the optimal filter combination
%for each $N$ 
without brute force search, we can use PCA analysis to find approximate solutions. 


This method initially creates one $3\times M$ matrix $\boldsymbol{M}$ and iteratively selects until reaching $k$ filters and thus our final $3 \times k$ matrix of weights. 
Specifically, the selection is performed by projecting $\boldsymbol{F}$ onto the space spanned by the target $\boldsymbol{X}$ matrix, finding the filter with the highest norm as per the Matching Pursuit method outlined in Mallat and Zhang \cite{mallat1993matching}.
We iteratively apply this methodology $k$ times, each time on a subset of the resultant orthonormal basis formed by subtracting the selected filter's projection.

This method is comparatively fast, able to compute the $k=5$ case in 19 seconds on a single L4 GPU. 
This method also scales well, with $k=8$ runtime at only 20 seconds.

\subsection{PCA Analysis with Macbeth Target}

For $k>3$, the original PCA Matching-Pursuit procedure can become weakly constrained because the target space induced directly by $\mathbf{X}\in\mathbb{R}^{3\times d}$ is only three-dimensional. To provide a richer objective, we instead construct a higher-dimensional target subspace using the Macbeth / X-Rite ColorChecker patch reflectances. Intuitively, this biases filter selection toward spanning the diversity of common surface reflectances, which often improves performance at larger $k$.

Let $\mathbf{R}\in\mathbb{R}^{P\times d}$ be a matrix of $P$ Macbeth patch reflectance spectra sampled on the same grid $\omega_d$ (with $P=24$ for a standard chart). We compute a thin SVD of $\mathbf{R}^\top$ and keep the top $r=\min(P,k)$ components to obtain a basis $\mathbf{U}_r\in\mathbb{R}^{d\times r}$. As before, we project candidate filters into this basis and greedily select indices by maximizing the norm of the component orthogonal to the span of previously selected projections.

\setlength{\floatsep}{6pt}
\setlength{\intextsep}{6pt}

\label{alg:pca-macbeth-selection}
\begin{algorithmic}[1]

\Ensure Selected indices $I\subset\{1,\dots,N\}$ with $|I|=k$ and weights $\boldsymbol{M}\in\mathbb{R}^{3\times k}$.
\State Compute thin SVD of Macbeth basis: $\mathbf{R}^\top = \mathbf{U}\boldsymbol{\Sigma}\mathbf{V}^\top$.
\State Set $r\leftarrow \min(P,k)$ and $\mathbf{U}_r \leftarrow \mathbf{U}(:,1\!:\!r)$.
\State Project candidates: $\mathbf{G} \leftarrow \mathbf{F}\,\mathbf{U}_r$ \Comment{$\mathbf{G}\in\mathbb{R}^{N\times r}$}
\State Initialize $I\leftarrow \emptyset$ and orthonormal basis $\mathbf{Q}\leftarrow [\ ]$.
\For{$t=1$ to $k$}
    \If{$t=1$}
        \State $i^\star \leftarrow \arg\max_i \|\mathbf{G}_{i,:}\|_2$.
        \State $\mathbf{q} \leftarrow \mathbf{G}_{i^\star,:}/\|\mathbf{G}_{i^\star,:}\|_2$.
    \Else
        \State $\widehat{\mathbf{G}} \leftarrow (\mathbf{G}\mathbf{Q})\mathbf{Q}^\top$.
        \State $\mathbf{G}_\perp \leftarrow \mathbf{G} - \widehat{\mathbf{G}}$.
        \State $s_i\leftarrow \|\mathbf{G}_{\perp,i,:}\|_2$ and force $s_i=-\infty$ for $i\in I$.
        \State $i^\star \leftarrow \arg\max_i s_i$.
        \State $\mathbf{q} \leftarrow \mathbf{G}_{\perp,i^\star,:}/\|\mathbf{G}_{\perp,i^\star,:}\|_2$.
    \EndIf
    \State $I \leftarrow I\cup\{i^\star\}$; $\mathbf{Q} \leftarrow [\mathbf{Q}\ \mathbf{q}]$.
\EndFor
\State Form selected matrix $\mathbf{F}_I\in\mathbb{R}^{k\times d}$ by stacking rows $i\in I$.
\State Fit weights to the original CMFs (MSE): $\boldsymbol{M} \leftarrow \mathbf{X}\,\mathbf{F}_I^{+}$.
\State \textbf{Optional:} refine $\boldsymbol{M}$ with gradient descent for non-MSE losses.
\end{algorithmic}
\end{algorithm}
\endgroup

In the original PCA Matching Pursuit as outlined above, we have an underdefined problem for $k>3$, as our 3 dimensional target space may be nearly spanned by the projection of the first 3 filters. 
One way to mitigate this issue is to project our space onto that as defined by the Macbeth Color Checker patches. 
Using this methodology, we define a higher-dimensional space and thus a lower incemental value per added filter, with a better result at high $k$. 
However, we will observe that this technique only accurately reproduces human eye color matching functions such as the CIE XYZ family of functions, and fails to generalize to other applications. 
That being understood, this method is able to achieve better metrics at high $k$. 
For practical purposes, the difference is immaterial due to the cost of increasing $k$ above 10 or so, in terms of size of components and the capture time increase inherent to adding additional filters.

\subsection{LASSO Selection}
To obtain a sparse (small-$k$) filter set without explicitly searching combinations, we also evaluate a LASSO-style selection procedure. Here we optimize a full weight matrix over the entire candidate set and use an $\ell_{2,1}$-type regularizer (group sparsity) to drive many filter-columns to near-zero magnitude.

Let $\mathbf{F}\in\mathbb{R}^{K\times d}$ denote the candidate filter-sensor rows and let $\mathbf{X}\in\mathbb{R}^{3\times d}$ denote the discretized target CMFs. We solve for a dense weight matrix $\boldsymbol{M}\in\mathbb{R}^{3\times K}$ by minimizing
\begin{equation}
    \min_{\boldsymbol{M}} \; L_d\!\left(\boldsymbol{M}\mathbf{F},\;\mathbf{X}\right) + \lambda \sum_{j=1}^{K}\left\|\boldsymbol{m}_{:,j}\right\|_2,
    \label{eq:lasso-group}
\end{equation}
where $\boldsymbol{m}_{:,j}$ is the $j$th column of $\boldsymbol{M}$. The second term promotes column-wise sparsity: when a column norm becomes small, the corresponding filter contributes negligibly to reconstruction.

After optimization, we rank filters by their column norms $\|\boldsymbol{m}_{:,j}\|_2$ and keep the top $k$ filters. Finally, we refit weights on the selected subset via a pseudo-inverse (least squares) to obtain the final $\boldsymbol{M}\in\mathbb{R}^{3\times k}$ used for evaluation.

\begingroup
\setlength{\textfloatsep}{6pt}
\setlength{\floatsep}{6pt}
\setlength{\intextsep}{6pt}
\begin{algorithm}[H]
\caption{LASSO / group-sparse filter selection}
\label{alg:lasso-selection}
\begin{algorithmic}[1]
\Require Target $\mathbf{X}\in\mathbb{R}^{3\times d}$; candidates $\mathbf{F}\in\mathbb{R}^{K\times d}$; desired number of filters $k$; regularization weight $\lambda$.
\Ensure Selected indices $I$ and final weights $\boldsymbol{M}_{I}\in\mathbb{R}^{3\times k}$.
\State Initialize $\boldsymbol{M}\leftarrow \mathbf{X}\,\mathbf{F}^{+}$ \Comment{Pseudo-inverse warm start}
\For{$t=1$ to $T$}
    \State $\widehat{\mathbf{X}} \leftarrow \boldsymbol{M}\mathbf{F}$
    \State $\mathcal{L} \leftarrow L_d(\widehat{\mathbf{X}},\mathbf{X}) + \lambda \sum_{j=1}^{K}\|\boldsymbol{m}_{:,j}\|_2$
    \State Update $\boldsymbol{M}$ by one Adam step on $\mathcal{L}$
\EndFor
\State Compute column norms $n_j\leftarrow \|\boldsymbol{m}_{:,j}\|_2$ for $j=1,\dots,K$
\State Select $I\leftarrow$ indices of the top-$k$ values of $n_j$
\State Form $\mathbf{F}_{I}\in\mathbb{R}^{k\times d}$ and refit $\boldsymbol{M}_{I} \leftarrow \mathbf{X}\,\mathbf{F}_{I}^{+}$
\end{algorithmic}
\end{algorithm}

\subsection{Brute Force Search Augmented PCA} 

$\;$\newline While brute force methods guarantee minimum possible error by investigating the entire search space, large $k$ implementations render this technique impractical. 
Additionally, our PCA method outlined above cannot always identify the optimal filter set, and has a nonnegligible MSE shortfall vs. exhaustive search for all $k>1$.

To address this problem, we use a blended method which finds a PCA solution for $k_1$ filters, and then considers all further possible combinations from a further $k_2$ filters, for a total $k=k_1+k_2$. 
This provides a middle ground to the performance-accuracy tradeoff.
There is negligible MSE difference between brute force and PCA methods at $k_2>1$ for all $k_1$.
Further, for $k_2<4$, the added time from the exhaustive search step is between 5-20 seconds on the 8 aforementioned L4 GPUs.

One challenging problem introduced by the extra choice of $k_2$ is an explosion of state space and a violation of certain earlier assumptions regarding brute force superiority. 
Whereas in the original problem, we have a monotonically decreasing relationship between MSE and $k$ in the brute force case (if the 'best' filter would decrease MSE, a weight of 0 would be applied), the introduction of the brute force rounds requires us to intellegently choose $k_2$. 
Then, under practical considerations in a filter wheel of limited $k$, the choice of $k_1$ vs. $k_2$ becomes a difficult to define and hard to optimize tradeoff.
In fact, certain artifacts emerge in the comparison between brute force and heuristic algorithmms with $k_2>1$, such as improved MSE performance for the PCA algorithm at $k=2$ as shown above. 

Presumably a brute force algorithm would necessarily find the global minimum MSE, however, with $k_2>0$ and the brute force solution 'branching' (in essence, considering $k_2$ filters from the $k_1$ brute force filters) may miss the optimal filter combination. 
Therefore, any brute force solution with $k_2>0$ should be considered with caution. 
There is value in augmenting a brute force solution, as runtime is greatly reduced, but we find that the effect of mixing search algorithms at an intermediate value provides an efficient yet accurate middle ground for practical values of $k$.
